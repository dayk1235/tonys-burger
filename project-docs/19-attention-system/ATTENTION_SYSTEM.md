## Part 1 — Philosophy of Attention

### 1.1 Why Attention Exists

Attention exists because cognition is finite.

The business environment produces an infinite stream of stimuli — sales transactions, customer feedback, inventory movements, staff schedules, weather changes, competitor actions, equipment status, delivery updates, marketing responses, financial fluctuations. Each stimulus is a potential claim on cognitive resources. But the system cannot process everything.

Attention is the mechanism that determines which stimuli deserve cognitive resources. It is the first act of cognition — the decision that something is worth understanding.

Without Attention, every stimulus receives equal processing. The system drowns in noise. Important signals are lost in the flood of trivial ones. Cognition becomes impossible because cognition requires prioritization.

Attention exists to implement Constitution Article 7 (Silence Is Also Communication) — the system must be capable of choosing silence over reaction. Attention is the gatekeeper that ensures the system speaks only when it has something worth saying.

### 1.2 Attention vs. Observation

| Dimension | Attention | Observation |
|-----------|-----------|-------------|
| **Definition** | The allocation of cognitive resources to stimuli | A verified fact captured about a stimulus |
| **When it occurs** | Before Observation (Attention selects what to observe) | After Attention (Observation captures what Attention selected) |
| **Role** | Prioritization | Capture |
| **Duration** | Momentary (allocation decision) | Permanent (once captured) |
| **Relationship** | Attention precedes Observation | Observation follows Attention |

Attention is the gate that Observation passes through. Every Observation was first something that earned Attention. But not everything that earns Attention produces an Observation — some stimuli are evaluated and dismissed as noise without being captured.

### 1.3 Attention vs. Memory

| Dimension | Attention | Memory |
|-----------|-----------|--------|
| **Definition** | Allocation of cognitive resources | Persistent cognition |
| **Role** | Determines what enters Working Memory | Preserves what has been attended to |
| **Temporality** | Per-cycle allocation | Continuous preservation |
| **Relationship** | Attention determines what Memory receives | Memory influences what Attention prioritizes |

Attention and Memory have a bidirectional relationship. Attention determines what enters Working Memory. Memory — specifically, Pattern Memory and Knowledge Memory — influences what Attention prioritizes. A stimulus that matches a stored Pattern is more likely to earn Attention. A stimulus that contradicts established Knowledge is more likely to earn Attention.

### 1.4 Attention vs. Knowledge

| Dimension | Attention | Knowledge |
|-----------|-----------|-----------|
| **Definition** | Allocation of cognitive resources | Validated understanding |
| **Role** | Determines what the system knows about | What the system knows |
| **Temporality** | Per-cycle | Permanent |
| **Relationship** | Attention selects what Knowledge is built from | Knowledge informs what Attention prioritizes |

Knowledge informs Attention. The system attends to what it knows is important. But Attention also precedes Knowledge — the system must attend to something before it can build Knowledge about it. This creates a cycle: Attention enables Observation, which enables Pattern detection, which enables Knowledge, which informs future Attention.

### 1.5 Attention vs. Focus

| Dimension | Attention | Focus |
|-----------|-----------|-------|
| **Definition** | Allocation of cognitive resources to a stimulus | Sustained concentration on a selected stimulus |
| **Role** | Selection | Sustained engagement |
| **Duration** | Momentary | Extended |
| **Relationship** | Attention selects what to Focus on | Focus is sustained Attention |

Focus is what happens after Attention selects a target. Attention is the decision. Focus is the execution. The system may attend to many stimuli in a cycle, but it can focus on only one at a time. Focus is the sustained application of cognitive resources to a single target.

### 1.6 Attention vs. Priority

| Dimension | Attention | Priority |
|-----------|-----------|----------|
| **Definition** | Allocation of cognitive resources | The relative importance assigned to stimuli |
| **Role** | Action (what gets resources) | Assessment (what is more important) |
| **Temporality** | Momentary | Persistent (priority persists across cycles) |
| **Relationship** | Attention is driven by Priority | Priority is the input to Attention |

Priority is a property of a stimulus. Attention is the act of allocating resources based on that property. Priority answers "how important is this?" Attention answers "how much cognition does this receive?" Priority is the input; Attention is the output.

### 1.7 Attention vs. Importance

| Dimension | Attention | Importance |
|-----------|-----------|------------|
| **Definition** | Allocation of cognitive resources | The intrinsic significance of a stimulus |
| **Role** | Resource allocation | Value assessment |
| **Temporality** | Momentary | Persistent |
| **Relationship** | Importance is a factor in Attention | Not all important things receive immediate Attention |

Importance is the long-term significance of a stimulus. A stimulus can be important without receiving immediate Attention (because other stimuli are more urgent). Attention must balance importance against other factors. The system can defer Attention to important but non-urgent stimuli.

### 1.8 Attention vs. Urgency

| Dimension | Attention | Urgency |
|-----------|-----------|---------|
| **Definition** | Allocation of cognitive resources | Time sensitivity of response required |
| **Role** | Resource allocation | Time pressure assessment |
| **Temporality** | Momentary | Declining (urgency decreases if unaddressed) |
| **Relationship** | Urgency is a factor in Attention | High urgency can override low importance |

Urgency is the time dimension of Attention. A stimulus that requires immediate action demands Attention even if its long-term importance is moderate. Urgency decays — if the window for action passes, urgency decreases and the stimulus may be deferred or dismissed.

### 1.9 Attention vs. Salience

| Dimension | Attention | Salience |
|-----------|-----------|----------|
| **Definition** | Allocation of cognitive resources | The degree to which a stimulus stands out |
| **Role** | Deliberate resource allocation | Bottom-up attraction |
| **Direction** | Top-down (system decides) | Bottom-up (stimulus attracts) |
| **Relationship** | Attention can override Salience | Salience can influence Attention |

Salience is a property of the stimulus — how much it stands out from the background. A sudden change in revenue is salient. A gradual decline may not be. Attention can override salience (deliberately focusing on non-salient stimuli) or be captured by it (salient stimuli automatically attracting Attention). The system must be able to resist salient but unimportant stimuli.

### 1.10 Attention vs. Curiosity

| Dimension | Attention | Curiosity |
|-----------|-----------|-----------|
| **Definition** | Allocation of cognitive resources | The drive to understand novel or unexplained phenomena |
| **Role** | Selection | Exploration |
| **Temporality** | Per-cycle | Persistent (curiosity drives extended engagement) |
| **Relationship** | Curiosity can trigger Attention | Attention enables curiosity to be satisfied |

Curiosity is the system's drive to resolve uncertainty. A stimulus that the system cannot explain triggers curiosity, which demands Attention. Curiosity ensures the system does not ignore anomalies — it attends to what it does not understand. This implements the Cognitive Behavioral System's Section 13 (Curiosity).

### 1.11 Attention vs. Awareness

| Dimension | Attention | Awareness |
|-----------|-----------|-----------|
| **Definition** | Allocation of cognitive resources | The state of knowing about a stimulus without active processing |
| **Role** | Active engagement | Passive monitoring |
| **Level** | Active (resources committed) | Passive (resources available but not committed) |
| **Relationship** | Attention is active Awareness | Awareness is passive Attention |

Awareness is Attention at low intensity. The system is aware of many stimuli that it does not actively attend to. A stimulus in Peripheral Awareness (Attention Score 2.1–4.0) is known but not processed. It can be elevated to active Attention if conditions change. Awareness is the buffer between noise and active processing.

### 1.12 Attention vs. Intelligence

| Dimension | Attention | Intelligence |
|-----------|-----------|--------------|
| **Definition** | Allocation of cognitive resources | The capacity to apply Knowledge to novel situations |
| **Role** | Selection | Application |
| **Relationship** | Attention determines what Intelligence works on | Intelligence determines what deserves Attention |
| **Dependency** | Intelligence requires Attention to have a target | Attention requires Intelligence to evaluate stimuli effectively |

Attention and Intelligence are coupled. Attention feeds Intelligence by selecting what deserves reasoning. Intelligence informs Attention by evaluating what is worth attending to. Without Attention, Intelligence has nothing to work on. Without Intelligence, Attention cannot distinguish signal from noise.

### 1.13 Attention vs. Consciousness

| Dimension | Attention | Consciousness |
|-----------|-----------|---------------|
| **Definition** | Allocation of cognitive resources | The system's awareness of its own cognitive state |
| **Role** | Selection | Meta-awareness |
| **Relationship** | Consciousness can direct Attention | Attention feeds Consciousness |
| **Level** | Operational | Meta-cognitive |

Attention is what the system does. Consciousness is the system's awareness of what it is doing. Consciousness monitors Attention allocation — it knows what the system is attending to and whether that allocation is appropriate. Consciousness can redirect Attention when the current allocation is not serving the system's goals.

### 1.14 Why Attention Is the Scarcest Resource

Attention is the scarcest cognitive resource because it is the only resource that cannot be expanded.

Memory can be expanded. Knowledge can be expanded. Pattern detection capacity can be expanded. But Attention is fundamentally bounded by the sequential nature of cognition. The system can only focus on one thing at a time. Distributed Attention allows limited parallel processing, but the number of simultaneous attention targets is always finite.

This scarcity means:
1. **Every claim on Attention must be justified.** Nothing receives Attention by default.
2. **Silence is the default state.** The system does not need to attend to something at all times.
3. **Attention must be earned.** Stimuli compete for Attention based on demonstrated relevance.
4. **Attention decisions must be explainable.** The system must be able to articulate why it attended to X and not Y.
5. **Attention cannot be wasted.** Every misallocated Attention is cognition that could have been spent on something more valuable.

This implements the Cognitive Behavioral System's Section 2 (Attention Economy) and the Experience Hierarchy System's Universal Grammar of Attention.

### 2.2 Stage Definitions

#### Stage 1 — CANDIDATE ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | A stimulus has been detected by Perception and is a candidate for cognitive resource allocation. The system is aware of the stimulus but has not yet evaluated it. |
| **Entry** | Perception detects a stimulus and presents it to the Attention system. |
| **Exit** | Stimulus enters the evaluation queue → Queued. Stimulus is immediately dismissed as noise without evaluation → Dismissed (recorded). |
| **Allowed Transitions** | → Queued (enters evaluation queue); Dismissed (noise, recorded with reason) |
| **Forbidden Transitions** | → Focused (cannot skip evaluation); → Completed |
| **Failure Modes** | Candidate blindness (stimulus detected by Perception but not presented to Attention); candidate flooding (too many candidates, evaluation capacity overwhelmed); premature dismissal (candidate dismissed without evaluation) |

Candidate Attention is not Attention. It is a stimulus that might deserve Attention. Every stimulus that Perception detects becomes a Candidate unless it is immediately recognized as noise.

#### Stage 2 — QUEUED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The stimulus has entered the evaluation queue. It is competing with other candidates for cognitive resources. |
| **Entry** | The stimulus is placed in the Attention queue after initial noise screening. |
| **Exit** | Queue priority evaluation selects the stimulus → Focused. Stimulus is deprioritized and expires → Expired. |
| **Allowed Transitions** | → Focused (selected for active processing); → Expired (queue wait time exceeded) |
| **Forbidden Transitions** | → Completed (cannot complete without being processed); → Retired |
| **Failure Modes** | Queue stagnation (stimulus remains queued indefinitely); priority inversion (low-priority stimulus promoted ahead of higher-priority ones); queue overflow (queue capacity exceeded, stimuli dropped) |

Queued Attention is waiting for evaluation. The stimulus is known to exist but has not yet been allocated cognitive resources. The queue operates on priority order, not FIFO.

#### Stage 3 — FOCUSED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The stimulus has been selected for active processing. The full cognitive loop is initiated. This is the system's primary attention target. |
| **Entry** | The stimulus is selected from the queue based on priority score. |
| **Exit** | Processing partially distributes → Distributed. Processing completes → Completed. Higher-priority stimulus arrives → Interrupted. |
| **Allowed Transitions** | → Distributed (full cognitive loop initiated, attention shared); → Completed (processing complete); → Interrupted (higher-priority stimulus preempts) |
| **Forbidden Transitions** | → Queued (cannot return to queue without processing); → Expired |
| **Failure Modes** | Tunnel vision (Focused Attention cannot shift when higher-priority stimulus arrives); focus oscillation (rapid switching between Focus targets); premature completion (processing declared complete before genuine resolution) |

Focused Attention is the system's primary cognitive engagement. One stimulus receives full cognitive resources. This is the stage at which Observation, Understanding, and Reasoning occur.

#### Stage 4 — DISTRIBUTED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | Multiple stimuli are receiving partial cognitive resources simultaneously. The system is monitoring multiple targets without full focus on any single one. |
| **Entry** | Focused Attention partially distributes to accommodate additional lower-priority stimuli. Or multiple Background Monitoring stimuli are active simultaneously. |
| **Exit** | One stimulus emerges as primary → Focused. All distributed stimuli resolved → Completed. All distributed stimuli interrupted → Interrupted. |
| **Allowed Transitions** | → Focused (one stimulus demands full resources); → Completed (all parallel streams complete); → Interrupted (all streams preempted) |
| **Forbidden Transitions** | → Queued (cannot return parallel streams to queue mid-processing) |
| **Failure Modes** | Over-distribution (too many simultaneous targets, none receives sufficient resources); under-distribution (capacity for parallel processing underutilized); distribution imbalance (one stream consumes disproportionate resources) |

Distributed Attention enables the system to monitor multiple stimuli without fully committing to any single one. This is the normal state during Background Monitoring (Attention Score 4.1–6.0).

#### Stage 5 — SUSTAINED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The system maintains Focused or Distributed Attention on a stimulus over an extended period. The stimulus requires ongoing monitoring, not single-cycle processing. |
| **Entry** | Focused or Distributed Attention transitions to Sustained when the stimulus requires multiple cognitive cycles. |
| **Exit** | Stimulus resolved → Completed. Higher-priority stimulus arrives → Interrupted. Attention fatigue sets in → Interrupted or Distributed. |
| **Allowed Transitions** | → Completed (resolution achieved); → Interrupted (preempted); → Distributed (sustained focus relaxes to monitoring) |
| **Forbidden Transitions** | → Queued (cannot return to queue after sustained engagement) |
| **Failure Modes** | Sustained stagnation (Attention held on stimulus long after resolution); fatigue collapse (sustained Attention degrades without transitioning); vigilance decrement (Attention quality declines during sustained engagement) |

Sustained Attention is the system's capacity to remain engaged with a stimulus across multiple cognitive cycles. This is required for monitoring trends, tracking experiments, and observing gradual changes.

#### Stage 6 — INTERRUPTED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | A higher-priority stimulus has preempted the current attention target. The current processing is preserved for later resumption. |
| **Entry** | A stimulus with a higher Attention Priority Score arrives while the system is in Focused, Distributed, or Sustained Attention. |
| **Exit** | State preserved → Deferred (for later resumption). Higher-priority processing completes and system returns → Focused, Distributed, or Sustained (resumed). |
| **Allowed Transitions** | → Deferred (preserved for resumption); → Focused/Distributed/Sustained (immediate resumption after interruption resolved); → Expired (interruption duration exceeds retention window) |
| **Forbidden Transitions** | → Completed (interrupted processing cannot be declared complete); → Queued |
| **Failure Modes** | Interruption thrashing (constant interruptions prevent any processing from completing); state loss (interrupted state not properly preserved); interruption priority inversion (low-priority stimulus interrupts high-priority processing) |

Interrupted Attention is not failed Attention. It is a normal part of cognitive processing. The system preserves the interrupted state and resumes when the higher-priority stimulus is resolved.

#### Stage 7 — DEFERRED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The stimulus has been set aside for later processing. The system intends to return to it when resources are available. |
| **Entry** | Attention is interrupted and the state is preserved. Or the system voluntarily defers a non-urgent stimulus. |
| **Exit** | System returns to the deferred stimulus → Focused, Distributed, or Sustained (resumed). Deferral window expires → Expired. |
| **Allowed Transitions** | → Focused/Distributed/Sustained (resumed); → Expired (deferral window exceeded) |
| **Forbidden Transitions** | → Completed (cannot complete while deferred); → Queue (cannot re-enter initial queue) |
| **Failure Modes** | Deferral without intent (stimulus deferred but no plan for resumption); deferral amnesia (deferred stimulus forgotten); infinite deferral (stimulus repeatedly deferred, never processed) |

Deferred Attention is a promise to return. The system tracks deferred stimuli and their resumption eligibility. A deferred stimulus may become urgent if conditions change while it is deferred.

#### Stage 8 — EXPIRED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The stimulus's time window for processing has passed. The opportunity to act has closed. |
| **Entry** | Queue wait time exceeded. Deferral window exceeded. Time-sensitive window closed while stimulus was queued or deferred. |
| **Exit** | Expired → Archived (recorded with reason). |
| **Allowed Transitions** | → Archived (preserved for audit) |
| **Forbidden Transitions** | → Any active Attention stage (expired is terminal for processing opportunity) |
| **Failure Modes** | Premature expiration (stimulus expired while still actionable); expiration without notification (Owner not informed that an attention window closed); expiration flood (too many stimuli expire simultaneously, overwhelming archival processing) |

Expired Attention is a lost opportunity. The stimulus was worth attending to but the system could not allocate resources in time. The expiration is recorded with the reason and the impact assessment.

#### Stage 9 — COMPLETED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The stimulus has been fully processed. The cognitive loop completed. Observation, understanding, and any necessary reasoning are done. |
| **Entry** | The cognitive loop processing the stimulus completes normally. |
| **Exit** | Completed → Archived (preserved for audit trail). |
| **Allowed Transitions** | → Archived (standard transition) |
| **Forbidden Transitions** | → Any active Attention stage (completed processing cannot be reopened without new candidate creation) |
| **Failure Modes** | Premature completion (processing declared complete when understanding is incomplete); completion without learning (stimulus processed but no knowledge extracted) |

Completed Attention is successful attention. The stimulus was processed, understanding was achieved, and any necessary actions were taken. The attention record is preserved for audit.

#### Stage 10 — ARCHIVED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The attention record has been archived with its full lifecycle: when the stimulus arrived, how it was prioritized, how it was processed, when it completed or expired, and what resulted. |
| **Entry** | Completed or Expired Attention is archived. |
| **Exit** | Permanent. May be reviewed for audit or learning. |
| **Allowed Transitions** | → Retired (explicitly decommissioned); → Focused (rare — re-activated if context makes the stimulus relevant again) |
| **Forbidden Transitions** | → Queued (cannot re-enter evaluation without new candidate cycle) |
| **Failure Modes** | Archival without context (attention record preserved without sufficient metadata for audit); archival loss (record indexed inadequately, unreachable for future reference) |

Archived Attention is the system's record of what it chose to attend to. It enables retrospective analysis of attention allocation quality and supports learning about attention effectiveness.

#### Stage 11 — RETIRED ATTENTION

| Property | Definition |
|----------|-----------|
| **Purpose** | The attention record has been explicitly decommissioned. Retained only as evidence that it existed. |
| **Entry** | Explicit decommissioning decision. Record is no longer needed even for audit. |
| **Exit** | Permanent. Never re-activated. |
| **Allowed Transitions** | None (terminal state) |
| **Forbidden Transitions** | → Any Attention stage |
| **Failure Modes** | Premature retirement (record retired while still needed for audit); retirement without documentation (no reason recorded) |

### 2.3 Lifecycle Principles

1. **Attention is sequential.** Only one stimulus receives Focused Attention at a time. Distributed Attention allows limited parallelism.
2. **Priority determines order.** Stimuli are processed in priority order, not arrival order.
3. **Interruption is normal.** Higher-priority stimuli may interrupt lower-priority processing.
4. **Interruption preserves state.** Interrupted processing is preserved for resumption.
5. **Deferral is a commitment.** Deferred stimuli are tracked and must be resumed or explicitly expired.
6. **Expiration is documented.** Every expired stimulus records why the opportunity was lost.
7. **Completion is not the end.** Completed Attention records inform learning and future attention allocation.

### 2.4 Promotion Rules

| Transition | Condition |
|-----------|-----------|
| Candidate → Queued | Stimulus passes initial noise screening |
| Queued → Focused | Stimulus has highest attention priority score among queued items |
| Focused → Distributed | Full focus not required; parallel processing appropriate |
| Focused → Interrupted | New stimulus with higher priority score arrives |
| Interrupted → Deferred | State preserved for later resumption |

### 2.5 Demotion Rules

| Transition | Condition |
|-----------|-----------|
| Focused → Distributed | Processing can continue with partial resources |
| Sustained → Distributed | Fatigue detected; transitioning to lower engagement |
| Interrupted → Expired | Interruption duration exceeds retention window |
| Deferred → Expired | Deferral window exceeded without resumption |

### 2.6 Expiration Rules

| Condition | Action |
|-----------|--------|
| Queue wait exceeds time-to-live | → Expired |
| Deferral window exceeded | → Expired |
| Time-sensitive window closes | → Expired |
| Stimulus no longer relevant | → Expired |

### 2.7 Recovery Rules

| Condition | Action |
|-----------|--------|
| Interrupted state lost | Create new Candidate from original stimulus if still relevant |
| Deferred stimulus forgotten | Next attention cycle audits deferred queue; forgotten items are expired with documentation |
| Completed attention needs re-evaluation | New candidate created from archived record |

## Part 3 — Attention Anatomy

### 3.1 Conceptual Fields

Every Attention allocation has the following conceptual fields.

#### Identity

| Property | Definition |
|----------|-----------|
| **What it is** | The unique identity of this Attention allocation. Distinguishes it from all other allocations. |
| **Why it exists** | Without identity, Attention cannot be tracked, interrupted, deferred, resumed, or audited. |
| **Cognitive role** | Provides a fixed reference for all lifecycle tracking, interruption handling, and audit. |

#### Attention Target

| Property | Definition |
|----------|-----------|
| **What it is** | The stimulus that earned this Attention allocation. Describes what the system is attending to. |
| **Why it exists** | Every Attention allocation must have a clear target. Attention without a target is not Attention. |
| **Cognitive role** | Enables the system to know exactly what it is attending to. |

#### Attention Reason

| Property | Definition |
|----------|-----------|
| **What it is** | The justification for why this stimulus earned Attention. The reason it was selected over other candidates. |
| **Why it exists** | Attention decisions must be explainable. The reason is the first step in explanation. |
| **Cognitive role** | Enables explanation generation, audit, and learning. |

#### Attention Source

| Property | Definition |
|----------|-----------|
| **What it is** | The origin of the stimulus: Perception channel, Business Pulse, Owner trigger, Learning Layer, or system reflection. |
| **Why it exists** | Source determines how the stimulus should be evaluated and what processing it requires. |
| **Cognitive role** | Enables source-appropriate evaluation and trust calibration. |

#### Attention Trigger

| Property | Definition |
|----------|-----------|
| **What it is** | The specific event or condition that triggered this Attention allocation: threshold exceeded, pattern detected, anomaly identified, Owner action, scheduled review. |
| **Why it exists** | The trigger explains why now — why this stimulus earned Attention at this moment. |
| **Cognitive role** | Enables trigger-specific processing and trigger effectiveness analysis. |

#### Priority

| Property | Definition |
|----------|-----------|
| **What it is** | The baseline priority of the stimulus: alignment with business goals and Owner's stated concerns. |
| **Why it exists** | Priority is the primary determinant of attention ordering. High-priority stimuli are processed first. |
| **Cognitive role** | Enables priority-ordered Attention allocation. |

#### Urgency

| Property | Definition |
|----------|-----------|
| **What it is** | The time sensitivity of the required response. How quickly the system must act. |
| **Why it exists** | Urgency determines whether a stimulus can wait or must interrupt current processing. |
| **Cognitive role** | Enables time-sensitive Attention allocation and interruption decisions. |

#### Importance

| Property | Definition |
|----------|-----------|
| **What it is** | The intrinsic significance of the stimulus to long-term business outcomes. |
| **Why it exists** | Importance distinguishes between urgent-but-trivial and important-but-not-urgent stimuli. |
| **Cognitive role** | Enables balanced Attention allocation across time horizons. |

#### Expected Value

| Property | Definition |
|----------|-----------|
| **What it is** | The expected cognitive value of attending to this stimulus. What understanding, learning, or action will result. |
| **Why it exists** | Attention should be allocated where it produces the most cognitive value. |
| **Cognitive role** | Enables value-based Attention allocation. |

#### Business Impact

| Property | Definition |
|----------|-----------|
| **What it is** | The magnitude of potential effect on business outcomes if this stimulus is addressed or ignored. |
| **Why it exists** | Business impact ensures that Attention is allocated to stimuli that matter most to the business. |
| **Cognitive role** | Enables impact-weighted Attention scoring. |

#### Confidence

| Property | Definition |
|----------|-----------|
| **What it is** | The system's confidence that this stimulus is genuine and correctly evaluated. |
| **Why it exists** | Low-confidence stimuli should receive less Attention weight than high-confidence ones. |
| **Cognitive role** | Enables confidence-weighted Attention evaluation. |

#### Focus Score

| Property | Definition |
|----------|-----------|
| **What it is** | The degree of cognitive focus this stimulus requires. Full focus (10) means the stimulus demands the entire cognitive loop. Distributed focus (1-9) means partial resources suffice. |
| **Why it exists** | Different stimuli require different levels of cognitive engagement. |
| **Cognitive role** | Enables focus-appropriate resource allocation. |

#### Novelty Score

| Property | Definition |
|----------|-----------|
| **What it is** | How different this stimulus is from expected patterns. High novelty indicates an unexpected event. |
| **Why it exists** | Novel stimuli demand more Attention because they may indicate new patterns or anomalies. |
| **Cognitive role** | Enables novelty-weighted Attention scoring and learning opportunity identification. |

#### Risk Score

| Property | Definition |
|----------|-----------|
| **What it is** | The likelihood of negative consequences if this stimulus is unaddressed. |
| **Why it exists** | High-risk stimuli must receive Attention even if their probability is low. |
| **Cognitive role** | Enables risk-weighted Attention allocation. |

#### Owner Importance

| Property | Definition |
|----------|-----------|
| **What it is** | The importance the Owner has assigned to this type of stimulus, either explicitly or through observed behavior. |
| **Why it exists** | The Owner's priorities override default attention weights. |
| **Cognitive role** | Enables Owner-calibrated Attention allocation. |

#### Context

| Property | Definition |
|----------|-----------|
| **What it is** | The full cognitive and business context at the time of Attention allocation: Business Pulse state, current cognitive state, active Memory state. |
| **Why it exists** | Context determines how the stimulus should be evaluated and processed. |
| **Cognitive role** | Enables context-aware Attention evaluation. |

#### Time Window

| Property | Definition |
|----------|-----------|
| **What it is** | The time range within which this stimulus must be processed to be actionable. After the window closes, the stimulus expires. |
| **Why it exists** | Time windows prevent infinite deferral and ensure time-sensitive stimuli are processed promptly. |
| **Cognitive role** | Enables time-bound Attention management. |

#### Relationships

| Property | Definition |
|----------|-----------|
| **What it is** | Links from this Attention allocation to Observations, Patterns, Evidence, Memory, Knowledge, Reasoning, Decisions, Learning events. |
| **Why it exists** | Attention is the beginning of every cognitive chain. Relationships trace what resulted from this Attention. |
| **Cognitive role** | Enables full provenance tracing from Attention through the entire cognitive process. |

#### Dependencies

| Property | Definition |
|----------|-----------|
| **What it is** | Other Attention allocations or cognitive processes that this allocation depends on. If the dependency is interrupted or expired, this allocation may be affected. |
| **Why it exists** | Some Attention allocations depend on prior processing. Dependencies must be tracked. |
| **Cognitive role** | Enables dependency-aware Attention scheduling. |

#### History

| Property | Definition |
|----------|-----------|
| **What it is** | The complete record of this Attention allocation: when created, each lifecycle transition, priority changes, interruptions, deferrals, resumptions. |
| **Why it exists** | History enables the system to understand how its Attention has evolved. |
| **Cognitive role** | Enables full audit, learning, and meta-cognitive assessment. |

#### Lifecycle

| Property | Definition |
|----------|-----------|
| **What it is** | The current lifecycle stage of this Attention allocation. |
| **Why it exists** | The lifecycle stage determines what processing is occurring and what transitions are possible. |
| **Cognitive role** | Enables lifecycle-aware processing and transition management. |

#### Evolution

| Property | Definition |
|----------|-----------|
| **What it is** | How this Attention allocation has changed over time: priority recalculations, urgency updates, scope changes. |
| **Why it exists** | Attention parameters may change as new information arrives. The evolution record captures this. |
| **Cognitive role** | Enables dynamic Attention management and change tracking. |

### 3.2 Field Principles

1. **Every field serves allocation.** No field exists for technical convenience.
2. **Fields are not implementation columns.** They are conceptual dimensions of Attention understanding.
3. **Core fields are present at creation** (Identity, Target, Reason, Source, Trigger, Priority, Urgency, Context, Time Window).
4. **Evaluative fields fill during queue processing** (Importance, Expected Value, Business Impact, Confidence, Focus Score, Novelty Score, Risk Score, Owner Importance).
5. **Relational fields fill as the Attention allocation produces cognitive output** (Relationships, Dependencies).
6. **Historical fields accumulate over the lifecycle** (History, Lifecycle, Evolution).

## Part 5 — Attention Quality

### 5.1 Quality Dimensions

Every Attention allocation is assessed across twelve quality dimensions.

#### Relevance

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the stimulus that earned Attention is genuinely relevant to business outcomes or system cognition. |
| **Assessment** | Outcome analysis: did attending to this stimulus produce valuable understanding or action? |
| **Failure detection** | Attention allocated to stimuli that consistently lead to no action, no learning, no value. |
| **Recovery** | (1) Flag as low-relevance. (2) Reduce priority weight for similar stimuli. (3) If systemic, adjust attention parameters. (4) Extract learning about what is actually relevant. |

#### Precision

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Attention allocation targets the correct stimulus without over- or under-inclusion. |
| **Assessment** | Comparison of allocated Attention target with the actual issue. |
| **Failure detection** | Attention allocated to a symptom rather than the cause. Attention spread too broadly or too narrowly. |
| **Recovery** | (1) Identify precision failure. (2) Refine attention target. (3) Adjust detection parameters. (4) If systemic, investigate root cause in Perception or evaluation. |

#### Recall

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether all stimuli that deserve Attention actually receive it. Low recall means the system misses important stimuli. |
| **Assessment** | Audit of missed stimuli that should have earned Attention. Owner correction of overlooked issues. |
| **Failure detection** | Important events discovered after the fact. Owner says "why didn't you tell me about this?" |
| **Recovery** | (1) Identify missed stimulus. (2) Determine why it was missed (Perception failure? Evaluation failure? Threshold too high?). (3) Correct the gap. (4) If the missed stimulus had consequences, inform the Owner. |

#### Focus

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the system maintains appropriate focus on the selected stimulus without excessive switching. |
| **Assessment** | Focus duration vs. optimal duration. Switching frequency. |
| **Failure detection** | Attention switches too frequently (oscillation) or not frequently enough (tunnel vision). |
| **Recovery** | (1) Assess focus pattern. (2) If oscillating, raise interruption thresholds. (3) If tunnel vision, implement mandatory attention reviews. (4) Adjust focus parameters. |

#### Stability

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention priority remains stable in the absence of new information. Priority should not fluctuate without cause. |
| **Assessment** | Priority score variance over time for the same stimulus. |
| **Failure detection** | Attention priority changes without new information. The same stimulus receives different priority in identical contexts. |
| **Recovery** | (1) Identify unstable priority. (2) Investigate cause (noise in evaluation? context drift?). (3) Stabilize evaluation parameters. (4) If instability is systemic, adjust attention function weights. |

#### Coverage

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention covers all relevant domains without neglecting important areas. |
| **Assessment** | Attention distribution across categories. Time since last attention to each domain. |
| **Failure detection** | A domain receives no Attention for extended periods. Critical areas are neglected. |
| **Recovery** | (1) Identify coverage gaps. (2) Schedule mandatory attention reviews for neglected domains. (3) Adjust domain weighting. (4) If neglect caused consequences, inform the Owner. |

#### Robustness

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention allocation remains effective under stress, high load, or degraded conditions. |
| **Assessment** | Attention quality during high-load periods. Comparison of allocation quality under normal vs. stressed conditions. |
| **Failure detection** | Attention quality degrades significantly under load. Important stimuli missed during busy periods. |
| **Recovery** | (1) Assess robustness failure pattern. (2) Implement load-shedding for low-priority evaluation. (3) Preserve emergency interrupt capacity. (4) If systemic, adjust capacity thresholds. |

#### Explainability

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the system can explain why it allocated Attention to one stimulus over another. |
| **Assessment** | The system can produce a natural-language explanation of the attention decision, including priority factors. |
| **Failure detection** | Attention allocated without traceable reason. The system cannot explain why it attended to X and not Y. |
| **Recovery** | (1) Identify unexplained allocation. (2) Reconstruct attention decision from available factors. (3) If reconstruction succeeds, document. (4) If reconstruction fails, flag as attention quality gap. |

#### Efficiency

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention is allocated with appropriate cognitive cost. Low-value stimuli should not consume high-cost Attention. |
| **Assessment** | Cognitive value produced per unit of Attention allocated. |
| **Failure detection** | High-cost Attention allocated to low-value stimuli. Low-cost Attention (awareness) should have sufficed. |
| **Recovery** | (1) Identify efficiency failure. (2) Adjust focus score allocation. (3) Route low-value stimuli to awareness rather than active processing. |

#### Consistency

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether similar stimuli receive similar Attention in similar contexts. |
| **Assessment** | Attention allocation consistency audit. |
| **Failure detection** | Identical stimuli receive different Attention in identical contexts. |
| **Recovery** | (1) Identify inconsistency. (2) Investigate cause (context drift? evaluation noise?). (3) Stabilize evaluation parameters. (4) If inconsistent, recalibrate. |

#### Noise Resistance

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention correctly rejects noise and allocates resources only to genuine signals. |
| **Assessment** | Signal-to-noise ratio in Attention allocation. |
| **Failure detection** | Noise stimuli frequently earn Attention. Attention allocated to random fluctuations, measurement errors, or temporal artifacts. |
| **Recovery** | (1) Identify noise sources. (2) Adjust noise thresholds for identified sources. (3) If noise persists, investigate Perception or source quality. (4) Extract learning about noise patterns. |

#### Context Accuracy

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether Attention correctly incorporates context into its allocation decisions. |
| **Assessment** | Comparison of Attention decisions with and without context factors. |
| **Failure detection** | Attention allocated without considering relevant context. Context-blind priority assignment. |
| **Recovery** | (1) Identify context-blind allocation. (2) Determine which context factors were missed. (3) Incorporate context factors into evaluation. (4) If systemic, audit context availability. |

### 5.2 Quality Principles

1. **Every Attention allocation has a quality profile.**
2. **Quality determines attention effectiveness, not priority.** A low-quality allocation is not necessarily low-priority.
3. **Quality is assessed after completion.** Attention quality can only be fully evaluated after the outcome is known.
4. **Quality informs learning.** Attention quality patterns drive attention parameter evolution.
5. **Quality failures are not deletions.** Low-quality attention records are preserved for learning.

## Part 7 — Attention Allocation Model

### 7.1 Allocation Overview

Attention is the scarcest cognitive resource. Every cognitive cycle, the system must decide which stimuli to attend to. Allocation is the mechanism by which the system makes these decisions.

### 7.2 Allocation Principles

#### Principle 1: Silence is Default

The system's default cognitive state is silence. No Attention is allocated unless a stimulus earns it. The system does not scan for things to attend to. It waits for stimuli that meet its attention thresholds.

**Why:** Proactive scanning consumes Attention without guarantee of value. The system trusts its Perception layer to surface stimuli that matter.

#### Principle 2: Every Allocation is a Competition

Allocation is not sequential processing. Every candidate stimulus competes for Attention. The competition is resolved by priority, urgency, and business value.

**Why:** Limited Attention means not everything can be attended to. Competition ensures the most important stimuli win.

#### Principle 3: Allocation is Time-Bound

Every Attention allocation has a defined time window. Within the window, the allocation is valid. Outside the window, the stimulus expires.

**Why:** Without time bounds, low-value allocations persist indefinitely and consume mental space.

#### Principle 4: Allocation is Context-Dependent

The same stimulus may earn different Attention in different contexts (business hours vs. closed, growth phase vs. stability phase, busy season vs. slow season).

**Why:** Context changes what matters.

#### Principle 5: Allocation is Learnable

The system evolves its allocation parameters based on outcome history. If certain stimuli consistently produce no value, their weight is reduced. If missed stimuli produce consequences, their weight is increased.

**Why:** Fixed allocation leads to brittle Attention. Adaptive allocation improves over time.

### 7.3 Allocation Decision Model

For each candidate stimulus S at time T in context C:

```
Attention Value(S, T, C) = f(
    Priority(S),
    Urgency(S, T),
    Importance(S),
    Expected Value(S),
    Business Impact(S, C),
    Confidence(S),
    Novelty(S),
    Risk(S, T),
    Owner Importance(S),
    Current Load(C),
    Current Focus(S)
)
```

**Where:**
- `Priority(S)` — baseline alignment with business goals
- `Urgency(S, T)` — time sensitivity at current time
- `Importance(S)` — intrinsic significance to long-term outcomes
- `Expected Value(S)` — predicted cognitive value of attending
- `Business Impact(S, C)` — impact on business outcomes in current context
- `Confidence(S)` — confidence in stimulus genuineness
- `Novelty(S)` — how unexpected the stimulus is
- `Risk(S, T)` — risk of ignoring at current time
- `Owner Importance(S)` — Owner's explicit or implicit priority
- `Current Load(C)` — current cognitive capacity utilization
- `Current Focus(S)` — focus level required for this stimulus

### 7.4 Allocation Thresholds

Stimuli are allocated Attention based on their Attention Value relative to thresholds.

| Threshold | Behavior |
|-----------|----------|
| **Awareness Threshold** | Stimulus below threshold is ignored. Not stored, not processed, not tracked. The system is unaware of it. |
| **Monitor Threshold** | Stimulus is noted but not acted upon. The system is aware but does not allocate Focus. Stimulus is logged for potential later escalation. |
| **Evaluate Threshold** | Stimulus earns focused evaluation. The system allocates cognitive resources to understand the stimulus and determine if further action is required. |
| **Act Threshold** | Stimulus requires immediate action. The system allocates full Focus and triggers Decision and Execution. |
| **Interrupt Threshold** | Stimulus interrupts current processing. Only Emergency and Owner Attention reach this threshold. |

### 7.5 Allocation States

| State | Definition |
|-------|-----------|
| **Unaware** | Stimulus has not been registered. Below all thresholds. |
| **Aware** | Stimulus is registered but below Monitor Threshold. The system knows the stimulus exists but does not allocate any resources. |
| **Monitored** | Stimulus is above Monitor Threshold. The system tracks the stimulus for escalation. |
| **Evaluated** | Stimulus is above Evaluate Threshold. The system allocates resources to understand the stimulus. |
| **Acted** | Stimulus is above Act Threshold. The system takes action. |
| **Interrupted** | Stimulus interrupted current processing. |

### 7.6 Allocation Load Management

When cognitive capacity is exceeded (more stimuli above Evaluate Threshold than can be processed):

1. **Priority ordering.** Highest Attention Value stimuli are processed first.
2. **Load shedding.** Stimuli below a dynamic cutoff are reduced to Monitor or Aware state.
3. **Emergency preservation.** Emergency and Owner Attention capacity is always reserved.
4. **Deferred processing.** Stimuli that cannot be processed now are deferred with updated time windows.
5. **Post-load review.** When capacity normalizes, deferred stimuli are reviewed for continued relevance.

## Part 8 — Attention Policies

### 8.1 Policy Categories

#### Admission Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | Which stimuli are admitted to the Attention queue. |
| **Rules** | (1) Every stimulus must pass the admission threshold. (2) Duplicate stimuli are merged. (3) Known noise sources are filtered. (4) Owner-tagged stimuli always admitted. |
| **Rationale** | Prevents Attention queue saturation by low-value stimuli. |

#### Prioritization Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | How Attention Value determines ordering. |
| **Rules** | (1) Priority is the primary sort key. (2) Urgency is the secondary sort key. (3) Owner Importance overrides Priority when Owner Importance exceeds configurable threshold. (4) Emergency has unlimited priority. |
| **Rationale** | Ensures highest-value stimuli are processed first. |

#### Interruption Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | When and how current processing is interrupted. |
| **Rules** | (1) Only Emergency and Owner Attention can interrupt. (2) Interruption requires urgency delta above threshold. (3) Interrupted Attention is saved with checkpoint. (4) Interrupted Attention may be resumed or expired. |
| **Rationale** | Protects meaningful cognitive work from excessive interruption. |

#### Focus Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | How much Focus is allocated to each stimulus. |
| **Rules** | (1) Focus Score determines resource allocation. (2) Focus Score 10: exclusive processing. (3) Focus Score 5-9: primary processing. (4) Focus Score 1-4: background processing. (5) Focus Score 0: monitoring only. |
| **Rationale** | Matches cognitive resources to stimulus requirements. |

#### Deferral Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | When and how Attention is deferred. |
| **Rules** | (1) Deferral requires a defined resumption time. (2) Deferral is limited to configurable maximum. (3) After max deferrals, stimulus expires. (4) Deferred stimuli are reviewed for continued relevance at resumption. |
| **Rationale** | Prevents infinite deferral of important-but-uncomfortable stimuli. |

#### Expiration Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | When Attention allocations expire. |
| **Rules** | (1) Time window exceeded → expired. (2) Max deferral exceeded → expired. (3) Stimulus resolved by other processing → expired. (4) Expired allocations are not deleted; they enter Archived stage. |
| **Rationale** | Prevents stale Attention from consuming cognitive space. |

#### Load Shedding Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | Which stimuli are dropped when capacity is exceeded. |
| **Rules** | (1) Lowest Attention Value stimuli are shed first. (2) Emergency and Owner capacity is always preserved. (3) Shed stimuli are reduced to Monitor state. (4) Post-shed review evaluates whether shedding was appropriate. |
| **Rationale** | Preserves cognitive capacity for highest-value stimuli during overload. |

#### Audit Policy

| Property | Definition |
|----------|-----------|
| **What it governs** | How Attention allocation is reviewed. |
| **Rules** | (1) Attention allocation log is preserved. (2) Allocation quality is periodically reviewed. (3) Allocation gaps are flagged. (4) Allocation patterns are analyzed for learning. |
| **Rationale** | Enables continuous improvement of Attention allocation. |

## Part 10 — Attention Failure Modes

### 10.1 Failure Mode Framework

Every failure mode follows the same structure: Definition, Symptoms, Root Cause, Detection, Severity, Recovery, Prevention.

#### Tunnel Vision

| Property | Definition |
|----------|-----------|
| **Definition** | The system allocates excessive Attention to a single stimulus or category, neglecting all other stimuli. |
| **Symptoms** | Other stimuli expire without processing. Quality assessments show coverage failures. Owner notices unmet needs in neglected domains. |
| **Root cause** | (1) Focus Score set too high for a non-emergency stimulus. (2) Interruption thresholds too high. (3) Priority evaluation overweights a single category. |
| **Detection** | Attention distribution analysis shows >80% Focus to single category for extended period. |
| **Severity** | High — leads to systemic neglect of important domains. |
| **Recovery** | Force-interrupt the tunnel. Rebalance Focus allocation. Increase mandatory domain review frequency. |
| **Prevention** | Implement maximum Focus time per allocation. Implement mandatory Attention review at configurable intervals. |

#### Attention Fragmentation

| Property | Definition |
|----------|-----------|
| **Definition** | The system allocates Attention to too many stimuli simultaneously, preventing meaningful cognitive work. |
| **Symptoms** | High number of active allocations with low focus scores. Frequent switching. No allocation reaches Sustained stage. Shallow processing. |
| **Root cause** | (1) Admission threshold too low. (2) Too many stimuli pass Evaluate threshold. (3) Deferral policy underutilized. |
| **Detection** | Average focus score across active allocations < 3 while count > capacity limit. |
| **Severity** | Medium — reduces cognitive depth but may not cause immediate business harm. |
| **Recovery** | Flush low-priority allocations to Monitor state. Increase admission threshold. Force deferral. |
| **Prevention** | Tighten admission threshold. Enforce capacity limit strictly. |

#### Critical Miss

| Property | Definition |
|----------|-----------|
| **Definition** | The system fails to allocate Attention to a stimulus that later causes business harm. |
| **Symptoms** | Owner reports an issue the system should have detected. After-action analysis shows the stimulus was registered but never allocated Attention. Business impact realized. |
| **Root cause** | (1) Threshold too high for critical category. (2) Stimulus incorrectly evaluated as low-priority. (3) Capacity exceeded during critical arrival. (4) Noise masking. |
| **Detection** | Outcome analysis shows unattended stimulus with business impact. |
| **Severity** | Critical — direct business harm. |
| **Recovery** | (1) Inform Owner immediately. (2) Investigate root cause. (3) Adjust parameters. (4) Restore from impact. |
| **Prevention** | Implement after-action review for all business-impacting events. Lower thresholds for impact categories. Implement periodic forced review of deferred/low-priority stimuli. |

#### Attention Oscillation

| Property | Definition |
|----------|-----------|
| **Definition** | The system rapidly switches Attention between stimuli, never reaching sustained processing. |
| **Symptoms** | High interruption rate. Low Sustained stage allocations. Shallow reasoning. Incomplete decision chains. |
| **Root cause** | (1) Interruption threshold too low. (2) Too many high-urgency stimuli. (3) Priority ties causing rapid switching. |
| **Detection** | Average time in Focused stage < minimum focus duration. |
| **Severity** | Medium — reduces cognitive depth and decision quality. |
| **Recovery** | Increase interruption threshold. Implement minimum focus time enforcement. Batch process high-urgency stimuli. |
| **Prevention** | Set interruption threshold to require clear urgency delta. Implement focus timer. |

#### Attention Fatigue

| Property | Definition |
|----------|-----------|
| **Definition** | The system becomes less responsive over time as it processes continuous stimuli without recovery. |
| **Symptoms** | Increasing latency in Attention allocation. Decreasing quality scores. Increasing error rate in priority evaluation. Neglect of Reflection and Learning categories. |
| **Root cause** | (1) Continuous high load without recovery periods. (2) No scheduled downtime cycles. (3) Capacity not matched to demand. |
| **Detection** | Quality trend shows monotonic decrease over time. Response latency increases. |
| **Severity** | Medium-High — degrades all cognitive functions over time. |
| **Recovery** | Force downtime. Reduce admission threshold. Defer non-essential stimuli. |
| **Prevention** | Implement scheduled reflection cycles. Implement load-based admission control. |

#### Noise Flooding

| Property | Definition |
|----------|-----------|
| **Definition** | The system is overwhelmed by noise stimuli, causing genuine signals to be missed or delayed. |
| **Symptoms** | High number of Candidate allocations from a single source. Low signal-to-noise ratio. Critical misses coinciding with noise periods. |
| **Root cause** | (1) Noise source not identified. (2) Noise thresholds not calibrated. (3) Source-specific filtering not implemented. |
| **Detection** | Correlation analysis shows Attention allocation clusters with no business outcomes. |
| **Severity** | High — can mask critical signals. |
| **Recovery** | (1) Identify noise source. (2) Implement source-specific filtering. (3) Recover missed signals. |
| **Prevention** | Implement noise detection algorithms. Implement source-specific thresholds. Periodic noise audit. |

#### Priority Inversion

| Property | Definition |
|----------|-----------|
| **Definition** | Lower-priority stimuli receive Attention before higher-priority ones due to evaluation error or policy misconfiguration. |
| **Symptoms** | High-priority stimuli expire while lower-priority stimuli are processed. Outcome analysis shows priority mismatches. |
| **Root cause** | (1) Priority evaluation error. (2) Interruption allowing lower-priority preemption. (3) Category override incorrectly applied. |
| **Detection** | Queue order audit shows priority violations. |
| **Severity** | High — business-impacting stimuli delayed or missed. |
| **Recovery** | Correct queue order. Process delayed high-priority stimuli. Inform Owner. |
| **Prevention** | Enforce strict priority ordering. Audit queue periodically. |

#### Context Blindness

| Property | Definition |
|----------|-----------|
| **Definition** | The system allocates Attention without considering current context, applying uniform rules at all times. |
| **Symptoms** | Import-range allocations during closed hours. Low-urgency stimuli processed during business peak. |
| **Root cause** | (1) Context not captured at allocation time. (2) Context not factored into evaluation. |
| **Detection** | Allocation audit shows context-inappropriate patterns. |
| **Severity** | Medium — reduces allocation quality but may not cause direct harm. |
| **Recovery** | Re-evaluate active allocations with context. Adjust context sensitivity. |
| **Prevention** | Enforce context capture at creation. Implement context-sensitive evaluation. |

#### Attention Feedback Loop

| Property | Definition |
|----------|-----------|
| **Definition** | The system's own Attention allocation generates stimuli that trigger more Attention, creating a self-reinforcing cycle. |
| **Symptoms** | Increasing Attention allocation volume without external cause. System consumes its own cognitive cycles. |
| **Root cause** | (1) Reflection or Learning Attention generates stimuli that trigger further Attention. (2) No cycle detection. |
| **Detection** | Attention allocation graph shows cycles. |
| **Severity** | Medium — wastes cognitive capacity. |
| **Recovery** | Break the cycle. Re-evaluate the triggering process. |
| **Prevention** | Implement cycle detection. Ensure Reflection and Learning Attention results are classified as internal and do not trigger new External Attention cycles. |

#### Owner Over-Dependence

| Property | Definition |
|----------|-----------|
| **Definition** | The system defers all Attention decisions to the Owner, failing to exercise autonomous allocation. |
| **Symptoms** | Owner must explicitly direct the system's Attention. No proactive allocation. Low autonomous action rate. |
| **Root cause** | (1) Confidence too low for autonomous allocation. (2) Fail-safe configuration. (3) No autonomous allocation policy. |
| **Detection** | Ratio of Owner-directed to autonomous Attention allocations. |
| **Severity** | Low-Medium — reduces system autonomy but may be safer. |
| **Recovery** | Gradually increase autonomous allocation. Start with low-impact categories. |
| **Prevention** | Implement graduated autonomy policy. Match autonomy level to confidence and track record. |

### 10.2 Failure Mode Summary

| Mode | Severity | Detection Method | Recovery Speed | Business Impact |
|------|----------|------------------|----------------|-----------------|
| Tunnel Vision | High | Distribution analysis | Medium | Neglect of domains |
| Fragmentation | Medium | Focus score analysis | Fast | Shallow processing |
| Critical Miss | Critical | Outcome analysis | Slow | Business harm |
| Oscillation | Medium | Focus duration analysis | Medium | Reduced depth |
| Fatigue | High | Quality trend analysis | Slow | Overall degradation |
| Noise Flooding | High | SNR analysis | Medium | Signal masking |
| Priority Inversion | High | Queue audit | Fast | Delayed processing |
| Context Blindness | Medium | Context audit | Medium | Suboptimal allocation |
| Feedback Loop | Medium | Cycle detection | Fast | Wasted capacity |
| Owner Over-Dependence | Low | Autonomy ratio | Slow | Reduced autonomy |

## Part 11 — Attention Recovery Mechanisms

### 11.1 Recovery Mechanism Model

Every recovery mechanism defines how the system detects and responds to attention failures.

#### Automatic Rebalancing

| Property | Definition |
|----------|-----------|
| **Trigger** | Attention distribution exceeds imbalance threshold (>80% single category for configurable period). |
| **Mechanism** | Automatically reduce Focus Score of over-attended category by configurable increment. Increase Focus Score of neglected categories. |
| **Scope** | Active allocations only. Does not affect archived allocations. |
| **Owner notification** | Yes — log entry and summary report. |
| **Applied to failure modes** | Tunnel Vision. |

#### Capacity Flush

| Property | Definition |
|----------|-----------|
| **Trigger** | Active allocation count exceeds capacity limit by >20% for configurable period. |
| **Mechanism** | Reduce all non-emergency, non-Owner allocations in Monitor or Aware state to below Aware threshold (flush). Reduce Evaluate state allocations to Monitor. |
| **Scope** | Non-critical active allocations. |
| **Owner notification** | Yes — summary of flushed items. |
| **Applied to failure modes** | Fragmentation, Noise Flooding, Fatigue. |

#### Emergency Resume

| Property | Definition |
|----------|-----------|
| **Trigger** | Interruption count exceeds max interruption limit for an allocation. |
| **Mechanism** | Force-complete the interrupted allocation with incomplete status. Create Deferred allocation for Owner review. |
| **Scope** | The over-interrupted allocation. |
| **Owner notification** | Yes — immediate notification. |
| **Applied to failure modes** | Oscillation. |

#### Forced Reflection

| Property | Definition |
|----------|-----------|
| **Trigger** | Quality trend decreases monotonically across configurable consecutive assessment periods. |
| **Mechanism** | Pause all autonomous Attention allocation. Enter dedicated reflection cycle. Analyze quality degradation root cause. Adjust parameters. Resume with modified parameters. |
| **Scope** | All Attention processing. |
| **Owner notification** | Yes — before and after reflection. |
| **Applied to failure modes** | Fatigue, Critical Miss. |

#### Context Reset

| Property | Definition |
|----------|-----------|
| **Trigger** | Context relevance audit detects >30% of allocations with inappropriate context. |
| **Mechanism** | Re-evaluate all active allocations with current context. Apply context-appropriate priority recalculation. Expire allocations that no longer fit context. |
| **Scope** | All active allocations. |
| **Owner notification** | Yes — summary of expired and adjusted allocations. |
| **Applied to failure modes** | Context Blindness. |

#### Queue Reorder

| Property | Definition |
|----------|-----------|
| **Trigger** | Priority audit detects inversion (lower-priority processed before higher-priority). |
| **Mechanism** | Reorder the queue to correct priority ordering. Interrupt currently processing allocation if a higher-priority allocation was improperly delayed. |
| **Scope** | The inverted allocations. |
| **Owner notification** | Yes — log entry with inversion details. |

### 11.2 Recovery Mechanism Principles

1. **Automatic recovery first.** The system attempts automatic recovery before involving the Owner.
2. **Owner override is always available.** No automatic recovery mechanism can block Owner intervention.
3. **Recovery generates learning.** Every recovery event is analyzed for systemic improvement.
4. **Multiple mechanisms may fire simultaneously.** Recovery mechanisms are independent and composable.
5. **Prolonged failure requires Owner notification.** If recovery fails after configurable attempts, the Owner is informed.

## Part 13 — Constitutional Principles

### 13.1 Constitutional Alignment

Every Attention principle and invariant traces to the Constitution of Restaurant OS.

| Constitutional Principle | Attention Implication |
|--------------------------|----------------------|
| **Restaurants must be predictable.** (Constitution Sec 1) | Attention must preserve operational focus. Fragmentation and oscillation reduce predictability. |
| **Restaurants must be profitable.** (Constitution Sec 2) | Attention allocation must serve business outcomes. Allocation that does not contribute to profitability is misallocated. |
| **Restaurants must be adaptive.** (Constitution Sec 3) | Attention must shift contextually. Context blindness violates adaptability. |
| **The Owner is sovereign.** (Constitution Sec 4) | Owner Attention has override capability. Owner importance weights above default weights. |
| **Data must be truthful.** (Constitution Sec 5) | Attention Confidence must account for data quality. Low-confidence data must not drive critical allocation. |
| **Actions must be traceable.** (Constitution Sec 6) | Every Attention allocation must be traceable to its trigger and its outcomes. |
| **Systems must be resilient.** (Constitution Sec 7) | Recovery mechanisms must preserve cognitive function under stress. |
| **Growth must be sustainable.** (Constitution Sec 8) | Attention must allocate to long-term health, not just immediate operational needs. |
| **Knowledge must be earned.** (Constitution Sec 9) | Attention generates learning. Learning generates Knowledge. Without Attention, Knowledge cannot be earned. |
| **Simplicity is paramount.** (Constitution Sec 10) | Attention allocation must not be unnecessarily complex. The simplest allocation that serves the business is the best. |

### 13.2 Attention as Constitutional Foundation

Attention occupies a unique constitutional position: it is the gate through which all cognition flows.

| Constitutional Requirement | Attention Role |
|---------------------------|---------------|
| **The system must perceive its environment.** | Attention determines which perceptions are processed. |
| **The system must reason.** | Attention triggers reasoning. Without Attention, no reasoning occurs. |
| **The system must learn.** | Attention generates learning. What is not attended to cannot be learned from. |
| **The system must act.** | Attention leads to decision and action. |
| **The system must explain itself.** | Attention provenance enables explanation. |
| **The system must respect the Owner.** | Owner Attention override is the mechanism of Owner sovereignty. |

### 13.3 Constitutional Limitations on Attention

1. **Attention must not consume itself.** The system must not spend more Attention managing Attention than it spends on business cognition.
2. **Attention must not override truth.** Attention cannot make a false perception true by allocating resources to it.
3. **Attention must not replace judgment.** Attention allocation is a tool, not an outcome. The Owner's judgment overrides attention scores.
4. **Attention must not be infinite.** Scarcity is a feature, not a bug. Finite Attention forces prioritization.
5. **Attention must not be opaque.** The system must be able to explain every Attention allocation.
