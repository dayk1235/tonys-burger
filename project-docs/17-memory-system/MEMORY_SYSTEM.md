## CONSTITUTIONAL STATEMENT

Memory is the system's persistent cognitive structure.

Memory is not storage. Storage implies passive retention of undifferentiated data. Memory is an active, organized, relational structure where every fact is connected to other facts, every memory has a purpose, and every retention has a cost.

Without Memory, every cognitive cycle starts from zero. Every Observation is a first encounter. Every Pattern is a first discovery. Every Decision is made without history. Learning is impossible because there is nothing to learn from.

Memory makes cognition cumulative. It is the reason the system gets smarter with every cycle.

## Part 1 — Philosophy of Memory

### 1.1 Why Memory Exists

Memory exists because cognition must persist.

An Observation exists at a point in time. A Pattern spans multiple Observations but exists only while Observations are being compared. Evidence exists only in relation to a claim. None of these survive the cognitive cycle that created them unless Memory preserves them.

Memory exists to implement Constitution Article 19 (Restaurant OS has a Memory), Article 20 (Restaurant OS Learns), and Article 21 (Learning is Cumulative). Without Memory, the system cannot remember what it learned yesterday, cannot detect trends across weeks, and cannot build on prior understanding.

Memory is the system's answer to: "What do we know? What have we seen before? What did we learn?"

### 1.2 Memory vs. Observation

| Dimension | Observation | Memory |
|-----------|-------------|--------|
| **Definition** | A verified fact captured at a point in time | Persistent cognition that spans time |
| **Temporality** | Instantaneous (captured at T) | Cumulative (built over T1..Tn) |
| **Independence** | Standalone (exists regardless of use) | Relational (exists only in connection to other cognition) |
| **Lifecycle** | Single capture, permanent record | Continuous evolution through promotion, decay, and retirement |
| **Example** | "Revenue on Tuesday was $4,287" | "Tuesdays are consistently the slowest day of the week" |

An Observation is a moment. Memory is a pattern across moments. The same Observation may enter Memory as a fact (Historical), contribute to a Pattern (Pattern Memory), or become part of Understanding (Long-Term Memory).

### 1.3 Memory vs. Evidence

| Dimension | Memory | Evidence |
|-----------|--------|----------|
| **Definition** | Persistent cognition | Justification connecting an artifact to a claim |
| **Role** | Retains and organizes experience | Supports or refutes claims |
| **Purpose** | Preservation | Justification |
| **Relationship** | Memory stores Evidence | Evidence is stored in Memory |
| **Dependency** | Memory does not require Evidence | Evidence always depends on Memory |

Evidence is what the system stores in Memory to justify its beliefs. Memory is where Evidence persists across time. Without Memory, Evidence would vanish the moment the claim was resolved.

### 1.4 Memory vs. Pattern

| Dimension | Memory | Pattern |
|-----------|--------|---------|
| **Definition** | Persistent cognitive structure | A recurring structure in Observations |
| **Role** | Preservation and organization | Discovery and description |
| **Temporality** | Continuous (always present) | Emergent (detected over time) |
| **Relationship** | Memory stores Patterns | Patterns are stored in Pattern Memory |
| **Dependency** | Memory exists without Patterns | Patterns require Memory (to store the Observations they span) |

Patterns are one of the things Memory preserves. Pattern Memory is a specialized subsystem that retains detected regularities. But Memory is broader — it preserves everything the system has experienced, not just the patterns within it.

### 1.5 Memory vs. Knowledge

| Dimension | Memory | Knowledge |
|-----------|--------|-----------|
| **Definition** | Persistent cognitive structure | Validated understanding |
| **Role** | Preserves all experience | Represents what is reliably true |
| **Validation** | Stores everything (including hypotheses) | Stores only what has been validated |
| **Relationship** | Knowledge is a subset of Memory | Knowledge lives in Knowledge Memory |
| **Confidence** | Memory items have variable confidence | Knowledge requires minimum confidence |

Knowledge is Memory that has been validated. All Knowledge is in Memory, but not all Memory is Knowledge. A tentative hypothesis in Short-Term Memory is Memory but not Knowledge. This distinction implements Constitution Article 5 (Knowledge Is Earned).

### 1.6 Memory vs. Learning

| Dimension | Memory | Learning |
|-----------|--------|----------|
| **Definition** | Persistent cognition | The process of changing cognition based on experience |
| **Role** | Preservation | Transformation |
| **Temporality** | Continuous | Event-driven |
| **Relationship** | Memory enables Learning (by preserving experience) | Learning updates Memory (by promoting validated understanding) |
| **Direction** | Memory → Learning (feed) | Learning → Memory (update) |

Memory and Learning form a cycle. Memory preserves experience, which enables Learning to extract understanding, which updates Memory with validated knowledge. Without Memory, Learning is impossible. Without Learning, Memory is static.

### 1.7 Memory vs. Recall

| Dimension | Memory | Recall |
|-----------|--------|--------|
| **Definition** | Persistent cognitive structure | The act of retrieving from Memory |
| **Role** | Preservation | Retrieval |
| **Temporality** | Always present | Episodic (triggered by need) |
| **Relationship** | Memory enables Recall | Recall accesses Memory |
| **Dependency** | Memory exists without Recall | Recall depends on Memory |

Recall is a cognitive act. Memory is the persistent state that makes Recall possible. The system does not remember everything it could recall — it recalls what is needed for the current cognitive task.

### 1.8 Memory vs. Context

| Dimension | Memory | Context |
|-----------|--------|---------|
| **Definition** | Persistent cognitive structure | The situational frame for current cognition |
| **Role** | Preservation across time | Grounding for current processing |
| **Temporality** | Spans past and present | Present-focused |
| **Relationship** | Memory provides context | Context constrains which Memory is relevant |

Context is the subset of Memory that is relevant to the current cognitive cycle. Context is not a separate system — it is Memory filtered by Attention.

### 1.9 Memory vs. History

| Dimension | Memory | History |
|-----------|--------|---------|
| **Definition** | Persistent cognitive structure | The objective record of past events |
| **Role** | Active cognition about the past | Passive record of the past |
| **Relationship** | Memory is how the system experiences History | History is what Memory preserves |
| **Selectivity** | Memory is selective | History is complete |

History is the complete, immutable record. Memory is the system's active relationship with that record. Historical Memory is the system's History.

### 1.10 Memory vs. Archive

| Dimension | Memory | Archive |
|-----------|--------|---------|
| **Definition** | Active cognitive structure | A preserved record no longer actively used |
| **Role** | Supports ongoing cognition | Preserves for future reference |
| **Access** | Actively indexed | Available but not indexed for routine use |

The Archive is a subset of Memory. Historical Memory is the system's archive. The system can always retrieve from the archive; it simply does not consult it during routine cognitive processing.

### 1.11 Memory vs. Attention

| Dimension | Memory | Attention |
|-----------|--------|-----------|
| **Definition** | Persistent cognitive structure | The allocator of cognitive resources |
| **Role** | Preservation | Prioritization |
| **Temporality** | Continuous | Per-cycle allocation |
| **Relationship** | Attention determines what enters Memory | Memory influences what Attention prioritizes |

Attention and Memory are deeply coupled. Attention determines what enters Working Memory. Memory influences what Attention prioritizes.

### 1.12 Why Restaurant OS Never Forgets Arbitrarily

Restaurant OS forgets deliberately, never arbitrarily.

Forgetting is governed by explicit rules:
1. **Decay.** Unreinforced memories lose confidence over time. But they are not deleted.
2. **Deprecation.** Knowledge that is superseded is archived with a deprecation record.
3. **Contradiction.** When new Evidence contradicts stored Memory, both are preserved.
4. **Capacity management.** Lower-priority content is demoted rather than deleted.

The system never silently deletes. It never overwrites. It never chooses between conflicting Memories by hiding one.

This implements Constitution Article 19 and Article 29.

## Part 2 — Memory Lifecycle

### 2.1 Lifecycle Model

Memory begins as a potential retention candidate and evolves through stages of active processing, promotion, archive, and eventual retirement.

```
                    ┌──────────────────────┐
                    │  POTENTIAL            │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Identified as worth
                               │ retaining
                               ▼
                    ┌──────────────────────┐
                    │  WORKING              │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Multiple cycles
                               │ confirm relevance
                               ▼
                    ┌──────────────────────┐
                    │  SHORT-TERM           │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Reinforced and
                               │ promoted
                               ▼
                    ┌──────────────────────┐
                    │  ACTIVE               │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Pattern detected
                               │ from content
                               ▼
                    ┌──────────────────────┐
                    │  PATTERN              │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Validated and
                               │ promoted
                               ▼
                    ┌──────────────────────┐
                    │  KNOWLEDGE            │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ No longer actively
                               │ referenced
                               ▼
                    ┌──────────────────────┐
                    │  HISTORICAL           │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Compressed and
                               │ indexed
                               ▼
                    ┌──────────────────────┐
                    │  ARCHIVED             │
                    │  MEMORY               │
                    └──────────┬───────────┘
                               │ Explicitly
                               │ decommissioned
                               ▼
                    ┌──────────────────────┐
                    │  RETIRED              │
                    │  MEMORY               │
                    └──────────────────────┘
```

### 2.2 Stage Definitions

#### Stage 1 — POTENTIAL MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | An Observation, Understanding, or intermediate cognitive artifact has been identified as potentially worth retaining beyond the current cognitive cycle. |
| **Entry** | The Attention Layer or Understanding stage flags the artifact as having potential value. |
| **Exit** | Artifact enters Working Memory → Working Memory. Artifact assessed as not worth retaining → dismissed. |
| **Allowed Transitions** | → Working Memory (retention candidate confirmed); Dismissed (recorded with reason) |
| **Forbidden Transitions** | → Short-Term Memory (cannot skip Working); → Active Memory |
| **Failure Modes** | Retention blindness; false retention; flooding |

#### Stage 2 — WORKING MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The artifact is actively being processed in the current cognitive cycle. |
| **Entry** | The artifact enters Working Memory from Potential or re-enters from Short-Term or Active Memory. |
| **Exit** | Cognitive cycle completes → cleared OR promoted to Short-Term Memory. |
| **Allowed Transitions** | → Short-Term Memory (worth tracking); Cleared (cycle completes) |
| **Forbidden Transitions** | → Knowledge Memory; → Historical Memory |
| **Failure Modes** | Overflow; premature clearing; retention without processing |

#### Stage 3 — SHORT-TERM MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The artifact has been identified as worth tracking across multiple cognitive cycles. |
| **Entry** | Promoted from Working Memory when Understanding identifies a potential pattern. |
| **Exit** | Reinforced → Active Memory. Not reinforced → discarded. |
| **Allowed Transitions** | → Active Memory (reinforcement confirmed); Discarded (window expires) |
| **Forbidden Transitions** | → Knowledge Memory; → Historical Memory |
| **Failure Modes** | Premature promotion; prolonged retention; false reinforcement |

#### Stage 4 — ACTIVE MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The artifact has been confirmed as relevant across multiple cycles. Actively indexed and readily available for reasoning. |
| **Entry** | Promoted from Short-Term Memory when reinforcement thresholds are met. |
| **Exit** | Pattern detected → Pattern Memory. Validated as truth → Knowledge Memory. No longer relevant → demoted. |
| **Allowed Transitions** | → Pattern Memory; → Knowledge Memory; → Short-Term Memory; → Historical Memory |
| **Forbidden Transitions** | → Working Memory |
| **Failure Modes** | Active stagnation; premature demotion; over-indexing |

#### Stage 5 — PATTERN MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | A Pattern has been detected from Active Memory content. |
| **Entry** | Pattern Detection identifies a recurring structure across Active Memory items. |
| **Exit** | Pattern validated → Knowledge Memory. Pattern invalidated → Historical with deprecation record. |
| **Allowed Transitions** | → Knowledge Memory; → Historical Memory; → Active Memory |
| **Forbidden Transitions** | → Working Memory; → Short-Term Memory |
| **Failure Modes** | Pattern pollution; pattern stagnation; pattern fragmentation |

#### Stage 6 — KNOWLEDGE MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The Memory has been validated as reliably true. It is Knowledge. |
| **Entry** | Promoted from Active Memory or Pattern Memory through Learning Layer validation. |
| **Exit** | Knowledge deprecated → Historical Memory. |
| **Allowed Transitions** | → Historical Memory; → Active Memory |
| **Forbidden Transitions** | → Working Memory; → Short-Term Memory; → Pattern Memory |
| **Failure Modes** | False promotion; knowledge stagnation; knowledge rigidity |

#### Stage 7 — HISTORICAL MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The Memory is no longer active but is preserved as a permanent record. |
| **Entry** | Demoted from any earlier stage when no longer needed for active cognitive processing. |
| **Exit** | Permanent. May be re-activated. |
| **Allowed Transitions** | → Archived Memory; → Active Memory (rare) |
| **Forbidden Transitions** | → Working Memory; → Short-Term Memory; → Knowledge Memory |
| **Failure Modes** | Historical loss; premature archiving; archival without context |

#### Stage 8 — ARCHIVED MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | Historical Memory contents have been compressed for efficient long-term preservation. |
| **Entry** | Historical Memory contents evaluated for compression. |
| **Exit** | Permanent. May be decompressed. |
| **Allowed Transitions** | → Retired Memory; → Historical Memory |
| **Forbidden Transitions** | → Any active Memory stage |
| **Failure Modes** | Compression loss; premature compression; irreversible compression |

#### Stage 9 — RETIRED MEMORY

| Property | Definition |
|----------|-----------|
| **Purpose** | The Memory has been explicitly decommissioned. |
| **Entry** | Explicit decommissioning decision. |
| **Exit** | Permanent. Never re-activated. |
| **Allowed Transitions** | None (terminal state) |
| **Forbidden Transitions** | → Any Memory stage |
| **Failure Modes** | Premature retirement; retirement without reason; mass retirement |

### 2.3 Lifecycle Principles

1. **Forward motion is the default.** Memory progresses toward deeper integration.
2. **Stages are not skipped.** Memory cannot go from Working to Knowledge Memory without passing through Active.
3. **Demotion is possible.** Memory may move backward one stage with explicit re-evaluation.
4. **Historical is the common destination.** All active Memory eventually reaches Historical unless explicitly retired.
5. **Retirement is deliberate and documented.** Nothing is retired without a reason.
6. **Re-activation is possible but rare.** Historical Memory may be re-activated under new conditions.
7. **Multiple copies may exist simultaneously.** The same content may exist in Active Memory and Historical Memory.

## Part 4 — Memory Classification

### 4.1 Classification Model

Every piece of Memory belongs to exactly one primary category and may belong to zero or more secondary categories.

#### Working Memory

| Property | Definition |
|----------|-----------|
| **Definition** | The system's immediate consciousness. Current cognitive cycle context. |
| **Retention strategy** | Volatile. Cleared at cycle completion. |
| **Recall strategy** | Immediate (contents are the current focus). |

#### Operational Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of business operations: sales data, service times, inventory levels. |
| **Retention strategy** | Active while relevant. Decays as operations move forward. |
| **Recall strategy** | High-frequency. Consulted every cognitive cycle. |

#### Semantic Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Conceptual knowledge about the domain: what a Restaurant is, Ubiquitous Language, invariants. |
| **Retention strategy** | Permanent. Updated only through ADR. |
| **Recall strategy** | Loaded at initialization. |

#### Pattern Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of detected patterns: recurring structures, regularities, correlations. |
| **Retention strategy** | Variable. Retained as long as valid. |
| **Recall strategy** | Medium-frequency. Consulted during new Observation processing and Reasoning. |

#### Evidence Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of Evidence: justifications connecting artifacts to claims. |
| **Retention strategy** | Active while the claim is active. |
| **Recall strategy** | On-demand. Consulted for justification and confidence evaluation. |

#### Decision Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of recommendations and decisions: what was recommended, decided, and resulted. |
| **Retention strategy** | Active decisions retained. Resolved decisions archived. |
| **Recall strategy** | On-demand. Consulted for similar situations. |

#### Narrative Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of narratives the system has constructed. |
| **Retention strategy** | Active while relevant to current understanding. |
| **Recall strategy** | Low-frequency. Consulted for narrative continuity. |

#### Learning Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of learning events: what was learned, from what outcome. |
| **Retention strategy** | Permanent. Learning is never forgotten. |
| **Recall strategy** | Low-frequency. Consulted during meta-cognition. |

#### Business Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory specific to a particular business: identity, structure, Owner preferences. |
| **Retention strategy** | Lifetime of the business relationship. |
| **Recall strategy** | Every cognitive cycle. |

#### Strategic Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of strategic context: long-term goals, competitive positioning. |
| **Retention strategy** | Long-term. Updated during strategic reviews. |
| **Recall strategy** | Low-frequency. Consulted during strategic reasoning. |

#### Historical Memory

| Property | Definition |
|----------|-----------|
| **Definition** | The complete, immutable record of everything the system has experienced. |

#### Behavioral Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of behavior patterns: how customers, staff, and Owner behave. |

#### Procedural Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of how things work: procedures, causal relationships, process flows. |

#### Context Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory of the conditions under which things happened. |
| **Retention strategy** | Active while conditions remain relevant. |

#### Composite Memory

| Property | Definition |
|----------|-----------|
| **Definition** | Memory synthesized from multiple lower-level Memory items. |

### 4.2 Classification Principles

1. **Every Memory item has exactly one primary category.**
2. **A Memory item may have multiple secondary categories.**
3. **Category determines lifecycle parameters.**
4. **Category may evolve.**
5. **Some Memory items exist simultaneously in multiple categories.**

## Part 5 — Memory Quality

### 5.1 Quality Dimensions

Every piece of Memory is assessed across ten quality dimensions.

#### Accuracy

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory correctly represents reality. |
| **Assessment** | Cross-validation with independent sources. |
| **Failure detection** | Contradiction with new Observations. |
| **Recovery** | (1) Flag for review. (2) Reduce confidence. (3) Create contradiction record. (4) Create corrected Memory. (5) Audit downstream reasoning. |

#### Availability

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory can be retrieved when needed. |
| **Assessment** | Retrieval success rate. |
| **Failure detection** | Memory cannot be found when relevant. |
| **Recovery** | (1) Assess indexing gaps. (2) Improve indexing. (3) Document availability constraints. |

#### Integrity

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory is internally consistent, complete, and uncorrupted. |
| **Assessment** | Field validation. Relationship integrity check. |
| **Failure detection** | Missing required fields. Broken relationship links. |
| **Recovery** | (1) Identify failure type. (2) Attempt repair. (3) If repair fails, demote to Historical. |

#### Recallability

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory can be effectively retrieved based on available cues. |
| **Assessment** | Recall success rate. Relevance of retrieved Memory. |
| **Failure detection** | Memory exists but is not retrieved when relevant. |
| **Recovery** | (1) Assess indexing. (2) Add additional dimensions. (3) Strengthen relationships. |

#### Freshness

| Property | Definition |
|----------|-----------|
| **Meaning** | How recently the Memory was created, reinforced, or verified. |
| **Assessment** | Time since last reinforcement. |
| **Failure detection** | Memory age exceeds freshness threshold. |
| **Recovery** | (1) Mark as stale. (2) Reduce confidence. (3) Trigger re-verification. |

#### Consistency

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory is consistent with itself and established Knowledge. |
| **Assessment** | Internal consistency check. Cross-reference with related Memory. |
| **Failure detection** | Memory contradicts established Knowledge. |
| **Recovery** | (1) Create contradiction record. (2) Attempt reconciliation. (3) If not reconciled, reduce confidence. |

#### Accessibility

| Property | Definition |
|----------|-----------|
| **Meaning** | Whether the Memory can be accessed given current cognitive state. |
| **Assessment** | Retrieval under current cognitive load. |
| **Failure detection** | Memory exists but cannot be retrieved due to load. |

## Part 8 — Memory Failure Modes

### 8.1 Memory Loss

| Property | Definition |
|----------|-----------|
| **Cause** | Memory content is unintentionally inaccessible. |
| **Detection** | Identity-based recall fails. |
| **Mitigation** | Redundant indexing. Relationship-based cross-referencing. |
| **Recovery** | (1) Identify loss scope. (2) Attempt reconstruction from Historical. (3) If successful, create new Memory. (4) Document the loss. |

### 8.2 Memory Pollution

| Property | Definition |
|----------|-----------|
| **Cause** | Low-quality, incorrect content accumulates in Memory. |
| **Detection** | Increasing noise in retrieval. Decreasing reasoning accuracy. |
| **Mitigation** | Quality thresholds. Regular audits. |
| **Recovery** | (1) Identify sources. (2) Audit low-quality Memory. (3) Demote or deprecate. (4) Clean downstream reasoning. |

### 8.3 Memory Drift

| Property | Definition |
|----------|-----------|
| **Cause** | Memory content gradually changes over time without explicit updating. |
| **Detection** | Content no longer matches originating Observations. |
| **Mitigation** | Regular re-verification. Freshness-based decay. |
| **Recovery** | (1) Identify drifted Memory. (2) Assess magnitude. (3) Re-verify or deprecate. |

### 8.4 Memory Fragmentation

| Property | Definition |
|----------|-----------|
| **Cause** | Memory exists in isolation without adequate relationships. |
| **Detection** | Memory items with zero or minimal relationships. |
| **Mitigation** | Minimum relationship requirements. Periodic audits. |
| **Recovery** | (1) Identify fragmented Memory. (2) Establish missing relationships. (3) If not possible, deprecate. |

### 8.5 False Recall

| Property | Definition |
|----------|-----------|
| **Cause** | The system recalls Memory that does not exist or is incorrect. |
| **Detection** | Recalled content contradicts direct Observations. |
| **Mitigation** | Provenance verification on recall. |
| **Recovery** | (1) Identify false recall. (2) Trace cause. (3) Correct. (4) Audit downstream reasoning. |

### 8.6 Context Loss

| Property | Definition |
|----------|-----------|
| **Cause** | Memory content is retained but context is lost. |
| **Detection** | Missing Creation Context or Temporal Scope. |
| **Mitigation** | Mandatory context fields at creation. |
| **Recovery** | (1) Identify deficient Memory. (2) Attempt reconstruction. (3) If not possible, reduce confidence. |

### 8.7 Duplicate Memory

| Property | Definition |
|----------|-----------|
| **Cause** | The same artifact stored as multiple distinct Memory items. |
| **Detection** | Multiple items with identical content. |
| **Mitigation** | Duplicate detection at creation. |
| **Recovery** | (1) Identify duplicates. (2) Merge relationships. (3) Deprecate copies. |

### 8.8 Memory Saturation

| Property | Definition |
|----------|-----------|
| **Cause** | A Memory system has too many items. |
| **Detection** | Retrieval latency increases. Relevance decreases. |
| **Mitigation** | Capacity management. Importance-weighted retention. |
| **Recovery** | (1) Identify saturated system. (2) Assess contents by priority. (3) Demote lowest-priority. |

### 8.9 Recall Failure

| Property | Definition |
|----------|-----------|
| **Cause** | The system needs a specific Memory but cannot retrieve it. |
| **Detection** | Identity recall succeeds but context recall fails. |
| **Mitigation** | Redundant indexing. Multi-path recall strategies. |
| **Recovery** | (1) Identify failure context. (2) Try alternative paths. (3) If fails, report as inaccessible. |

### 8.10 Premature Forgetting

| Property | Definition |
|----------|-----------|
| **Cause** | Memory is demoted or archived before it should have been. |
| **Detection** | Later reasoning identifies needed Memory no longer active. |
| **Mitigation** | Conservative demotion thresholds. |
| **Recovery** | (1) Identify prematurely forgotten content. (2) Re-activate from Historical. (3) Adjust parameters. |
